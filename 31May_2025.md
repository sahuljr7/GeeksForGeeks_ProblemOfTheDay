Here's the **most optimized Java solution** for the problem **‚ÄúKth smallest element in a sorted matrix‚Äù** from GFG.

üîß **Approach**: **Binary Search on Value Range** (Optimal for sorted rows and columns matrix)

---

### ‚úÖ **Pseudocode Summary**

1. Let `lo = matrix[0][0]` and `hi = matrix[n-1][n-1]`
2. Binary search on value range `lo..hi`
3. For each mid value, count how many elements are `‚â§ mid` using efficient row-wise traversal.
4. Adjust `lo` or `hi` based on count
5. Stop when `lo == hi`, return `lo`

**Time Complexity:** `O(n * log(max - min))`
**Space Complexity:** `O(1)` ‚Äî constant space, no extra data structures used

---

### ‚úÖ **Final Java Code**

```java
class Solution {
    public int kthSmallest(int[][] matrix, int k) {
        int n = matrix.length;
        int low = matrix[0][0];
        int high = matrix[n - 1][n - 1];

        while (low < high) {
            int mid = low + (high - low) / 2;
            int count = countLessEqual(matrix, mid, n);

            if (count < k) {
                low = mid + 1;
            } else {
                high = mid;
            }
        }
        return low;
    }

    private int countLessEqual(int[][] matrix, int target, int n) {
        int count = 0;
        int row = n - 1;
        int col = 0;

        while (row >= 0 && col < n) {
            if (matrix[row][col] <= target) {
                count += row + 1;
                col++;
            } else {
                row--;
            }
        }
        return count;
    }
}
```

### ‚úÖ Why This Is Optimal

* **Avoids** full matrix flattening or sorting (`O(n¬≤ log n)`)
* **Avoids** heap usage (`O(k log n)`)
* Efficiently leverages matrix's **sorted rows and columns**
